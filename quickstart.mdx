This guide walks you through the minimum configuration required to start using Frappe HR: setting up HR-specific defaults, creating a holiday list and department, and adding your first employee with a leave policy.

<Note>
You need an Administrator or HR Manager account to complete these steps. If you have not installed Frappe HR yet, see the [Installation](/installation) guide first.
</Note>

<Steps>
  <Step title="Log into Frappe HR">
    Open your Frappe HR site in a browser:

    - **Docker / local bench**: [http://localhost:8000](http://localhost:8000) or [http://hrms.localhost:8080](http://hrms.localhost:8080)
    - **Frappe Cloud**: use the site URL from your dashboard

    Log in with your Administrator credentials. After login, click the grid icon in the top-left navbar and select **Frappe HR** to open the HR module at `/desk/people`.
  </Step>

  <Step title="Configure HR Settings">
    **HR Settings** is a single doctype that controls system-wide HR behaviour. Navigate to it via:

    > HR > Settings > HR Settings

    The settings are organised into tabs. Review and adjust the following key fields:

    **Employee tab**
    - **Employee Naming By** — choose how employee IDs are generated: `Naming Series`, `Employee Number`, or `Full Name`. The default is `Naming Series`.
    - **Standard Working Hours** — enter the number of working hours per day used for attendance calculations (for example, `8`).
    - **Retirement Age (In Years)** — set the default retirement age for your jurisdiction.

    **Leaves tab**
    - **Leave Approver Mandatory In Leave Application** — enabled by default. When checked, employees must select a leave approver before submitting a leave request.
    - **Send Leave Notification** — enabled by default. Uses the **Leave Approval Notification Template** and **Leave Status Notification Template** fields (pre-populated by the installer with default email templates).
    - **Restrict Backdated Leave Application** — enable this to prevent employees from applying for leave in the past. If enabled, set **Role Allowed to Create Backdated Leave Application** to the role that should be exempt (for example, `HR Manager`).

    **Shift and Attendance tab**
    - **Allow Employee Checkin from Mobile App** — enabled by default. Controls whether employees can check in via the Frappe HR PWA.
    - **Allow Geolocation Tracking** — enable to capture GPS coordinates on employee check-ins.

    Click **Save** when done.
  </Step>

  <Step title="Set up your company's Holiday List">
    A **Holiday List** defines the non-working days for a calendar year. Frappe HR uses it to calculate leave balances and payroll.

    Navigate to:

    > HR > Leaves > Holiday List > New Holiday List

    Fill in the following fields:

    | Field | Example value |
    |---|---|
    | **Holiday List Name** | `India Holidays 2026` |
    | **From Date** | `01-01-2026` |
    | **To Date** | `31-12-2026` |
    | **Weekly Off** | `Saturday`, `Sunday` |

    Use the **Get Weekly Offs** button to auto-populate all weekly off days for the year. You can also click **Get Holidays From** to import regional public holidays for your country.

    Add any additional public holidays manually in the **Holidays** table. Each row requires a **Date** and a **Description** (for example, `Republic Day`).

    Save the Holiday List. To make it the default for your company, open your company record (**Accounts > Masters > Company**), go to the **HR & Payroll** tab, and set the **Default Holiday List** field.
  </Step>

  <Step title="Create your first Department">
    Departments group employees and define approvers for leaves, expenses, and shift requests.

    Navigate to:

    > HR > Masters > Department > New Department

    Fill in:

    | Field | Example value |
    |---|---|
    | **Department Name** | `Engineering` |
    | **Company** | your company name |
    | **Leave Block List** | leave blank unless you want to block specific days |
    | **Payroll Cost Center** | optional — link to a cost center for payroll accounting |

    In the **Approvers** section, add at least one row to the **Leave Approver** table. Each row is a **User** who can approve leave applications from employees in this department.

    Save the Department.
  </Step>

  <Step title="Create your first Employee record">
    Navigate to:

    > HR > Employees > Employee > New Employee

    Complete the mandatory fields:

    | Field | Notes |
    |---|---|
    | **Employee Name** | Full name of the employee |
    | **Company** | Select your company |
    | **Date of Joining** | Employment start date |
    | **Department** | Select the department you just created |
    | **Employment Type** | `Full-time`, `Part-time`, `Contract`, etc. (fixtures created on install) |
    | **Designation** | Job title or role |
    | **Date of Birth** | Used for birthday reminders |
    | **Gender** | Required for payroll and compliance reports |

    In the **Approvers** section (added by Frappe HR as a custom section on the Employee doctype), set:

    - **Leave Approver** — the user who approves this employee's leave applications
    - **Expense Approver** — the user who approves expense claims
    - **Shift Request Approver** — the user who approves shift change requests

    Optionally, in the **Salary** section, set:

    - **Holiday List** — overrides the company default for this specific employee
    - **Default Shift** — the employee's regular shift type
    - **Grade** — links to an **Employee Grade** for salary structure lookups

    Click **Save**. Frappe HR assigns an employee ID based on the **Employee Naming By** setting in HR Settings.
  </Step>

  <Step title="Assign a Leave Policy">
    A **Leave Policy** specifies how many days of each leave type an employee is entitled to per year. Assigning it creates leave allocations automatically.

    First, create a Leave Policy if one does not exist:

    > HR > Leaves > Leave Policy > New Leave Policy

    Add rows to the **Leave Policy Details** table. Each row maps a **Leave Type** (for example, `Casual Leave`, `Sick Leave`, `Privilege Leave`) to an **Annual Allocation** count.

    Save the Leave Policy, then create a **Leave Policy Assignment**:

    > HR > Leaves > Leave Policy Assignment > New Leave Policy Assignment

    | Field | Value |
    |---|---|
    | **Employee** | the employee you just created |
    | **Leave Policy** | the policy you created above |
    | **Assignment based on** | `Leave Period` or leave blank to set dates manually |
    | **Effective From** | start of the leave period |
    | **Effective To** | end of the leave period |
    | **Carry Forward** | check to carry over unused leaves from the previous period |

    Click **Save**, then click **Allocate Leaves**. Frappe HR creates one **Leave Allocation** record per leave type defined in the policy.
  </Step>
</Steps>
